# candle-labeler

A human-in-the-loop tool for labeling forex/gold (or any OHLCV
instrument) chart data with **buy / neutral / sell** decisions across
multiple timeframes at once - while an online-learning PyTorch model
watches every decision, trains on it immediately, and tracks its own
predictions alongside yours for comparison.

Each labeling round, you're shown the *same* decision point through
**six different lookback windows** (300, 200, 100, 50, 20, and 10
candles back) as six separate browser tabs, so you can judge the same
moment at multiple zoom levels before committing to a call on each one.

---

## How it works, at a glance

1. `main.py` picks a random candle from your historical data.
2. It opens 6 browser tabs (via `chart_app.py`), each showing a
   candlestick chart ending at that candle, with a different amount of
   history behind it (300/200/100/50/20/10 candles).
3. Each chart also shows a **horizontal volume profile** on the right
   edge, so you can see where volume concentrated by price level, not
   just over time.
4. You click **Buy**, **Neutral**, or **Sell** in each tab. The tab
   closes itself once you've answered.
5. Once all 6 are answered, your decisions are appended as one row to
   `my_data.csv`, and fed to the online model (`model_agent.py`),
   which:
   - First predicts what it *would* have guessed for each timeframe
     (printed for comparison against your actual choice).
   - Then trains on your labeled decision and auto-saves its updated
     weights to disk.
6. You're asked whether to run another round or quit.

## File-by-file explanation

### `main.py`
The main loop. Loads the candle history, initializes the model,
launches 6 chart processes per round, waits for all 6 decisions,
appends them to `my_data.csv`, runs a model prediction + training step,
and loops until you enter `0` to quit.

⚠️ **Careful:** `main.py` calls `kill_other_python_processes()` at the
start of every round, which kills **every other Python process on your
machine**, not just ones related to this project - it's there to make
sure old chart windows don't linger and hog ports 8051-8056. Don't run
this on a machine where you have other important Python processes
running.

### `chart_app.py`
Renders a single interactive Dash/Plotly candlestick window. Shows the
volume profile, lets you click the chart to draw a temporary reference
line, and the Buy/Neutral/Sell buttons that report your decision back
to `main.py` through a shared dict and close the tab.

### `model_agent.py`
The `OnlineModelAgent` class: a small multi-head PyTorch network (one
shared feature extractor, six separate output heads - one per
timeframe). Each labeled example is:
1. Turned into a 7-number feature vector describing the window's price
   action (return, volatility, where the close sits in its range,
   candle body strength, volume trend, etc.) instead of a flat average.
2. Standardized using a running per-timeframe normalizer.
3. Stored into a persistent replay buffer, and a mini-batch (including
   the newest example) is used for each training step - so the model
   learns from a spread of past examples each round, not just the
   latest one.
4. Checkpointed to `model_dir/` after every round.

### `output_file.py`
Data-prep utilities, used by `setup_data.py`:
- `convert_to_candles(input_file, output_file, seconds)` - converts raw
  tick/second-level data into fixed-interval OHLCV candles.
- `create_csv_file(filename)` - creates the empty `my_data.csv` with
  the correct header.
- `add_row(...)` - a manual helper for appending a row outside the main
  app loop (not used by `main.py` itself).

### `setup_data.py`
**Run this exactly once**, before ever running `main.py`. It calls the
two `output_file.py` functions above to build your candle file and
your empty labeled-data file. See "Getting started" below.

## Getting started

### 1. Install dependencies

```bash
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Add your data

Place your raw tick/second-level price CSV into `data/`. See
[`data/README.md`](data/README.md) for the exact expected format. This
repo does not include any price data - you provide and hold the rights
to your own data.

### 3. Run the one-time setup

Open `setup_data.py`, update the `input_file` path to match your raw
data filename, then run:

```bash
python setup_data.py
```

This builds `data/xauusd_1min.csv` and creates the empty `my_data.csv`.
Example of what it's doing under the hood:

```python
from output_file import convert_to_candles, create_csv_file

# Converts raw tick data into 1-minute candles.
# Raw format:   timestamp,open,high,low,close,volume
convert_to_candles("data/xauusd_s1_20260529_20260628.csv", "data/xauusd_1min.csv", 60)

# Creates the file that will store your trading labels.
# Format: candle_number,timestamp,open,high,low,close,volume  (for the candle file)
create_csv_file("my_data.csv")
```

### 4. Run the labeling app

```bash
python main.py
```

Six browser tabs will open per round. Click Buy / Neutral / Sell in
each, and the app will log your decision, train the model, and ask if
you want to run another round.

## Notes / known behavior

- The model checkpoint format is tied to its feature dimensionality. If
  you change the feature extraction in `model_agent.py` (add/remove
  features), any existing `model_dir/online_model.pth` will fail to
  load with a shape-mismatch error - this is caught and handled by
  falling back to a freshly initialized model, with a printed warning.
- Loss values reflect a mini-batch average (not a single example), so
  expect a moderate, gradually-declining loss as you label more data -
  not a fast drop to near-zero every round.
- `"neurall"` (typo) and `"neutral"` are both treated as the same class
  throughout - this was kept intentionally for backward compatibility
  with existing data/fallback values rather than renamed everywhere.
